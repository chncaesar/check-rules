## 规则：串口/UART 通信禁用 JSON，必须用二进制协议 + CRC

**场景**：MCU/嵌入式设备通过串口（UART/serial）收发指令或数据。

**做**：用二进制协议，固定帧结构（帧头 + 长度 + 负载 + CRC16 校验）；接收方按帧解析。

**禁**：在串口上发 JSON 文本（如 `{"cmd":"start"}`）；用分隔符模式分帧。

**原因**：串口易丢字节、易粘包，JSON 文本无法可靠分帧和校验；二进制帧 + CRC 才能检测损坏。

**例**：发启动指令用二进制帧「帧头 + 命令字 + CRC16」，不是发 `{"cmd":"start"}` 字符串。

## 规则：串口收发必须有超时和重试

**场景**：串口请求-响应交互。

**做**：设置接收超时（如 500ms）；失败重试（如 3 次）。

**禁**：无超时地阻塞等待响应。

**原因**：设备可能无响应，无超时会永久挂起。
